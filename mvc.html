Архітектура MVC (Model-View-Controller)

Архітектура MVC (Model-View-Controller) - це шаблон проектування, який розділяє додаток на три основні логічні компоненти: модель, представлення та контролер. Це робиться для окремого управління різними аспектами додатка.

Структура MVC

  • Модель (Model): Модель представляє структуру даних. Вона відповідає за збереження, відновлення та обробку даних для додатка.
  • Представлення (View): Представлення відображає дані, тобто візуалізує модель даних для користувача.
  • Контролер (Controller): Контролер обробляє вхідні запити користувачів. Він взаємодіє з моделлю та вибирає представлення для відображення даних.

Переваги MVC

  • Поділ відповідальності: Кожен компонент в архітектурі MVC має свою власну відповідальність, що спрощує управління додатком.
  • Полегшена підтримка: Оскільки різні частини додатка розроблені незалежно одна від одної, це полегшує підтримку та оновлення додатка.
  • Гнучкість: Завдяки розділенню на модель, представлення та контролер, розробники можуть працювати паралельно на різних частинах додатка без взаємного впливу.
            
            // Controller (Activity or Fragment)
            public class MyController extends Activity {
                private MyModel model;
                private MyView view;
    
                @Override
                protected void onCreate(Bundle savedInstanceState) {
                    super.onCreate(savedInstanceState);
                    model = new MyModel();
                    view = new MyView(this, model);
                    setContentView(view);
                }
            }
    
            // Model
            public class MyModel {
                private String data;
    
                public String getData() {
                    return data;
                }
    
                public void setData(String data) {
                    this.data = data;
                }
            }
    
            // View
            public class MyView extends View {
                private MyModel model;
    
                public MyView(Context context, MyModel model) {
                    super(context);
                    this.model = model;
                }
    
                @Override
                protected void onDraw(Canvas canvas) {
                    super.onDraw(canvas);
                    // Use the model data to draw something
                }
            }
